# Project Product Design - Checklist

> 产品设计项目管理检查清单

---

## 项目创建检查

### 文档结构检查

```
projects/{name}/
├── Project_Goal.md          ✓ 用户创建，AI 只读
├── progress.md              ✓ 初始化当前阶段
├── decisions.md             ✓ 创建空文件
├── input_product_doc.md     （可选）已有产品文档
└── output_product_doc.md    ✓ 创建空文件（最终产出）
```

### 初始化确认

- [ ] Project_Goal.md 已由用户编写并包含：项目背景、目标用户、核心目标
- [ ] progress.md 已初始化，当前阶段设为"阶段 1：明确项目背景和目标"
- [ ] 明确告知用户：Project_Goal.md 由用户维护，AI 不会修改

---

## 阶段推进检查

### 阶段 1 → 阶段 2：需求拆解

- [ ] 已阅读 Project_Goal.md 理解需求
- [ ] 已生成两个任务列表：
  - [ ] 需求文档任务列表（以功能点为单位的 PRD 编写任务）
  - [ ] 原型页面设计任务列表（以页面为单位的设计任务）
- [ ] progress.md 已更新为"阶段 2：拆解需求和功能点"

### 阶段 2 → 阶段 3：PRD 编写

- [ ] 任务列表已写入 progress.md
- [ ] 当前聚焦的任务已明确
- [ ] progress.md 已更新为"阶段 3：编写需求文档"

### 阶段 3 → 阶段 4：结构验证

- [ ] 已生成"整体信息架构总览"
- [ ] 功能模块划分已检查确认
- [ ] progress.md 已更新为"阶段 4：验证需求结构"

### 阶段 4 → 阶段 5：原型设计

- [ ] 已完成 PRD 编写
- [ ] progress.md 已更新为"阶段 5：推进原型设计"

### 阶段 5 设计任务检查（每任务）

- [ ] 已调用 `prototype-prompt-generator` 生成 Figma Make Prompt
- [ ] 用户已完成设计并提供设计方案链接
- [ ] **【强制】** 设计方案链接已记录到 progress.md
- [ ] 任务状态已从"🔄 设计中"更新为"✅ 已完成"

### 阶段 5 → 阶段 6：文档汇总

- [ ] 所有设计任务已完成
- [ ] 已生成 output_product_doc.md
- [ ] 每个功能包含：用户故事、业务规则、验收标准
- [ ] **【关键文档待 Review】** 已提示用户 Review output_product_doc.md

### 阶段 6 → 阶段 7（或完成）

- [ ] 用户已 Review 并通过 output_product_doc.md
- [ ] 如有需求变更，进入阶段 7
- [ ] 如无变更，项目标记为完成

---

## 需求变更检查（阶段 7）

当需求发生变更时：

- [ ] decisions.md 已更新，记录变更原因
- [ ] 受影响的需求文档任务状态已改为"需求变更，待更新"
- [ ] 受影响的原型页面设计任务状态已改为"需求变更，待更新"
- [ ] 已提示更新产品文档
- [ ] 已给出原型修改建议（增量更新而非重新生成）

---

## 每轮对话结束检查

### 文档更新检查

- [ ] progress.md 已更新当前阶段和任务进度
- [ ] 如有新决策，decisions.md 已记录
- [ ] 如有需求变更，已按阶段 7 处理

### 交付物检查

- [ ] 设计任务标记为"✅ 已完成"时，必须有设计方案链接
- [ ] 禁止：交付物为"待补充"或空值时标记完成

### 用户确认检查

- [ ] 如有待确认内容，已明确提示用户
- [ ] 如有关键文档待 Review，已阻塞并等待确认

---

## 项目完成检查

- [ ] output_product_doc.md 已完成并通过用户 Review
- [ ] progress.md 状态已更新为"🟢 已完成"
- [ ] 所有设计任务都有设计方案链接
- [ ] decisions.md 已归档所有关键决策

---

## 常见错误规避

| 错误 | 正确做法 |
|------|----------|
| 修改 Project_Goal.md | **严禁**，只读，如需修改提示用户 |
| 设计任务无链接就标记完成 | **强制要求**设计方案链接 |
| 需求变更不更新相关任务 | 同步更新所有受影响任务状态 |
| 跳过阶段直接设计 | 按 1-7 阶段顺序推进 |
| output_product_doc.md 缺少用户故事 | 确保每个功能都有设计原因 |
